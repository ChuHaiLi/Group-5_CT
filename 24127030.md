# Task 1 – Thiết Kế và Thu Thập Dữ Liệu Mô Phỏng (Data Collection)

## 1. Problem Definition (Xác định vấn đề)
Mục tiêu của task là tạo ra một **bộ dữ liệu tĩnh (sample dataset)** mô phỏng danh sách **địa điểm du lịch** để hệ thống có thể dùng làm đầu vào cho phần gợi ý thông minh (Recommendation).

Bộ dữ liệu cần:
- Bao gồm **10–15 địa điểm du lịch tiêu biểu ở Việt Nam**.
- Dạng **file JSON**, dễ tích hợp vào Flask hoặc API backend.
- Mỗi mục gồm các trường:
  - `name`: tên địa điểm
  - `type`: loại hình (park, museum, tour, beach, v.v.)
  - `price`: giá vé hoặc chi phí tham quan
  - `rating`: điểm đánh giá trung bình
  - `tags`: thẻ mô tả
  - `lat`, `lon`: tọa độ (phục vụ bản đồ)
  - `trend_score`: điểm xu hướng (phục vụ AI gợi ý)

**Kết quả mong đợi:** file `sample_places.json` có cấu trúc rõ ràng, được kiểm tra cú pháp hợp lệ và có thể load lên hệ thống Flask mà không lỗi.

---

## 2. Decompose (Phân rã bài toán)

| Bước | Nội dung công việc | Kết quả đầu ra |
|------|---------------------|----------------|
| **B1. Xác định cấu trúc dữ liệu** | Quyết định các trường thông tin cần có (name, type, price, rating, tags, lat, lon, trend_score). | Mẫu cấu trúc JSON hoàn chỉnh. |
| **B2. Thu thập dữ liệu thực tế** | Chọn 15 địa điểm du lịch nổi bật ở Việt Nam (ví dụ: Hồ Gươm, Bến Nhà Rồng, Đà Lạt Flower Park, v.v.). | Danh sách địa điểm thô. |
| **B3. Chuẩn hóa và mô phỏng dữ liệu** | Điền dữ liệu vào file JSON theo đúng cấu trúc; thêm `trend_score` mô phỏng mức độ phổ biến. | File `sample_places.json` hoàn chỉnh. |
| **B4. Kiểm tra tính hợp lệ** | Kiểm tra định dạng JSON trên [jsonlint.com](https://jsonlint.com/) hoặc bằng Flask test. | File JSON hợp lệ, có thể parse được. |
| **B5. Kiểm thử với Flask** | Viết đoạn code Flask (file test.py) đọc file sample_places.json và in dữ liệu ra console. | Dữ liệu hiển thị đúng, không lỗi. |
